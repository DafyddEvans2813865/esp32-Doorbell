LED Blink Troubleshooting

Quick checklist
- Confirm upload succeeded (PlatformIO shows "SUCCESS" in terminal).
- Ensure correct pin: match code pin number to physical GPIO (e.g., 17).
- Verify wiring: LED anode -> resistor -> GPIO pin; LED cathode -> GND.
- Confirm LED polarity: long leg = anode (+), short = cathode (−).
- Use 220–330Ω resistor in series with LED.
- Ensure `pinMode(pin, OUTPUT);` is present in `setup()`.

Step-by-step checks
1) Re-check `src/main.cpp`
- Ensure the pin constant matches: `const int LED_PIN = 17;`
- Minimal test loop:

#include <Arduino.h>
const int LED_PIN = 17;
void setup(){ pinMode(LED_PIN, OUTPUT); }
void loop(){ digitalWrite(LED_PIN, HIGH); delay(200); digitalWrite(LED_PIN, LOW); delay(200); }

2) Try the onboard LED
- Replace `LED_PIN` with `LED_BUILTIN` and upload. If this blinks, the board and code work; external wiring is suspect.

3) Check pin capabilities and boot functions
- Some pins have special roles at boot or are input-only. Verify GPIO17 is usable on your board (check board docs).
- Avoid pins used for flash/boot unless you know their behavior.

4) Observe serial/logs
- Add serial prints to confirm `setup()` runs:

Serial.begin(115200);
Serial.println("Setup complete");

- Use PlatformIO Monitor:
```bash
pio device monitor
```

5) Quick hardware tests
- Swap LED to a different GPIO (e.g., 2 or 4) and update code.
- Test the LED with a 3V coin cell or multimeter continuity/voltage to confirm it lights.

6) Power & cable
- Ensure USB cable supplies data+power (some power-only cables exist).
- Confirm board is powered during upload and run.

7) Build & upload commands (project root)
```bash
pio run
pio run --target upload
pio device monitor
```

8) If upload succeeds but no blink
- Try toggling the pin state very fast (delay 100ms) to make brief flashes easier to spot.
- Try a bare-bones sketch that sets pin HIGH once in `setup()` to test steady output.
